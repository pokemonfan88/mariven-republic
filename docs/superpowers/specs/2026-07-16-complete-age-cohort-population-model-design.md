# 完整年龄队列人口模型设计

**日期：** 2026-07-16

**状态：** 已实现（schema v3）

**范围：** 全国单岁年龄—性别队列、出生、全因死亡、迁入迁出、年龄推进、人口公开指标、状态迁移与主 Tick 集成

## 1. 目标

将当前固定的顶层 `population` 数值升级为可跨日持久化、可复现、可解释的完整人口账本，并为登革热、犯罪、教育、就业、医疗、GDP 和人均指标提供统一人口基础。

模型必须满足：

- 以 2026-08-11 的 1,200,000 人作为权威基准，不反推或伪造此前人口流量。
- 内部按男性、女性和 0 至 100 岁以上单岁年龄维护人口。
- 第一滚动 365 天在无重大超额死亡冲击时严格达到 1.9% 净增长。
- 每日出生、死亡和迁移为整数，并有确定性波动，而不是固定日均值。
- 全因死亡只从人口中扣除一次；事故、疾病和灾难模型只能分类死亡或明确增加超额死亡。
- 相同输入状态重复 Tick 得到完全相同的输出。
- 中途 JSON 序列化、重新加载后继续运行，与连续运行完全相同。
- 所有内部队列之和始终严格等于顶层 `population`。
- 运行时不依赖网络或第三方 Python 库。

## 2. 已确认人口口径

### 2.1 基准日期与首周期

- 基准状态日期：`2026-08-11`
- 基准常住人口：`1_200_000`
- 第一周期第一个 Tick 日期：`2026-08-12`
- 第一周期最后一个 Tick 日期：`2027-08-11`
- 第一周期长度：365 天
- 无超额死亡情景的期末人口：`1_222_800`
- 第一周期净增长：`22_800`，即基准人口的 `1.9%`

2026 年人口年报只包含模型启用后的部分年度，并明确标记 `partial_year=true`。从 2027 年开始可以形成完整日历年度报告。滚动 365 天指标不受此限制。

### 2.2 第一周期流量账本

| 流量 | 365 天配额 |
|------|-----------:|
| 出生 | 27,500 |
| 全因基线死亡 | 6,600 |
| 侨民永久回流 | 2,500 |
| 外国移民流入 | 2,200 |
| 马里文人永久移出 | 2,800 |
| 总迁入 | 4,700 |
| 净移民 | +1,900 |
| 自然增长 | +20,900 |
| 总净增长 | +22,800 |

恒等式为：

```text
27,500 - 6,600 + 2,500 + 2,200 - 2,800 = 22,800
```

### 2.3 长期路径

第一周期之后，不再硬编码每年总增长率：

- 总和生育率从 2026 年的 2.3 缓慢下降，到 2035 年约为 2.0。
- 年龄结构的变化自然改变育龄女性人数和出生数。
- 男女死亡率沿生命表缓慢改善，并允许未来医疗、疾病和灾难模型施加有来源的变化。
- 疫情后侨民回流逐步减弱。
- 外国移民和永久移出继续作为独立流量。
- 长期人口增长率预计向世界观中的约 0.8% 靠近，但不强制每年命中 0.8%。

## 3. 数据来源与基线构建

### 3.1 先验来源

年龄—性别曲线采用斐济单一先验：

- 联合国《World Population Prospects 2024》单岁、分性别人口与生命表数据：<https://population.un.org/wpp/>
- 斐济统计局年龄—性别人口结构与人口指标：<https://www.statsfiji.gov.fj/statistics/social-statistics/population-and-demographic-indicators/>

斐济数据只提供先验形状。马里文最终基线不得直接照搬斐济人口数、总增长率、生育率或寿命。

### 3.2 马里文校准锚点

基线必须校准到：

- 总人口：1,200,000
- 中位年龄：27.8 岁
- 总和生育率：2.3
- 年出生：约 27,500
- 男女综合预期寿命：69.2 岁
- 男性预期寿命：67.0 岁
- 女性预期寿命：71.5 岁
- 婴儿死亡率：每千活产 16.2
- 五岁以下死亡率：每千活产 21.3
- 全因死亡：约 6,600 人/年

全因死亡 6,600 人来自现有死因统计的内部一致性：心血管死亡约 1,200 人，占全部死亡 18.2%，反推总死亡约 6,593 人；设计采用整数校准值 6,600。

## 5. 主 Tick 数据流

每日固定顺序：

```text
天气
  ↓
汇率与商品
  ↓
CPI
  ↓
显著事件与显著死亡
  ↓
人口队列结算
  ├─ 月初年龄推进（仅每月第一个 Tick）
  ├─ 当日出生
  ├─ 显著死亡年龄—性别分配
  ├─ 补足其他全因死亡
  ├─ 明确超额死亡
  ├─ 侨民回流
  ├─ 外国移民
  └─ 永久移出
  ↓
状态验证
  ↓
归档
```

人口模型使用独立的 `population` 随机流和命名子流。人口模型的加入不得改变天气、汇率、商品、CPI 或事件模型既有随机结果。

## 6. 年龄队列结构

### 6.1 单岁、分性别队列

内部年龄组：

```text
male[0] ... male[99], male[100+]
female[0] ... female[99], female[100+]
```

共 202 个公开可验证的内部队列。所有值均为非负整数。

### 6.2 生日月份桶

为避免每年 1 月 1 日集体老化，也避免简单“每月转移十二分之一”导致同一人口一年内跨越多个年龄，内部为每个年龄—性别队列保存 12 个生日月份桶。

每月第一个 Tick 在出生、死亡和迁移之前推进该月生日桶：

- 1 月第一个 Tick 推进 1 月生日人口。
- 2 月第一个 Tick 推进 2 月生日人口。
- 依此类推。
- 推进按年龄从高到低处理。
- 99 岁进入 100+ 开放组。
- 100+ 人口不再离开最终年龄组。
- 年龄推进不改变总人口。

出生人口按出生月份进入 0 岁对应生日桶。由于当月年龄推进先于当月出生，新生儿不会在出生当月被错误推进为 1 岁。迁入人口按确定性月份分布进入相应生日桶；死亡和迁出同时从年龄队列及其生日桶扣除。

基准状态记录 `last_aging_month="2026-08"`，表示 2026 年 8 月年龄推进已包含在 2026-08-11 基线中。下一次年龄推进发生在 2026-09-01。按月推进是汇总模型近似，生日统一落在月初，但不会改变年度年龄结构或造成一年内重复跨龄。

`cohorts` 是生日月份桶的汇总。验证必须保证二者逐年龄、逐性别相等。

## 7. 平衡随机配额

### 7.1 周期计划

每个滚动 365 天周期开始时，模型为以下五类流量生成完整日计划：

- 出生
- 基线全因死亡
- 侨民回流
- 外国移民
- 永久移出

计划生成步骤：

1. 为周期内每个日期生成独立确定性权重。
2. 将季节因子与命名随机扰动相乘。
3. 对 365 个权重归一化。
4. 用最大余数法把年度整数配额分配到每一天。
5. 通过稳定的日期排序和随机流处理余数并列。
6. 把五类完整日计划写入状态。

这样每日值存在自然波动，周期合计又严格等于配额。中途保存与恢复不会改变未来计划。

### 7.2 季节性边界

第一版只采用温和且有依据的季节性：

- 出生：允许小幅月份差异，不改变年度总量。
- 侨民永久回流：12 月至 1 月权重较高，但临时返乡者不计入常住人口。
- 外国移民：可以保持近似平稳。
- 永久移出：学年和招聘周期可以产生小幅波动。
- 基线死亡：按月份长度和温和季节因子分布；疾病模型尚未接入前不得伪造具体疫情。

季节因子必须在规格化测试中有明确上限，不能制造不可信的单日尖峰。

## 8. 出生模型

第一周期出生总额严格为 27,500。

- 新生儿进入 0 岁队列。
- 出生性别比采用约 105 名男婴对应 100 名女婴。
- 第一周期的男女出生总额也通过配额计划保持稳定比例，而不是每日独立硬币抽样。
- 15 至 49 岁分年龄生育率使用斐济曲线作为形状，再缩放到马里文总和生育率和出生总量。
- 第一周期之后，出生配额根据当时育龄女性人数、分年龄生育率和长期 TFR 路径生成。
- 经济与教育模型未来只能通过版本化输入修改下一周期生育率，不在第一版中实现短期经济冲击。

第一版不模拟孕产过程、双胞胎、婚姻状态或家庭单位。

## 9. 死亡账本与去重

### 9.1 单一权属

人口模型拥有全国人口会计和全因死亡。事件、疾病和灾难模型：

- 可以解释或分类全因死亡。
- 可以明确添加超额死亡。
- 不得在人口模型之外直接扣减人口。
- 同一死亡不得被多个模型重复扣除。

### 9.2 每日结算

人口模型先接收事件模型生成的显著死亡数量。显著死亡类别包括现有的交通、溺水、自杀、谋杀、工伤、雷击和其他显著事件。

若当天基线全因死亡计划为 18 人，显著死亡为 2 人：

```text
全因死亡             18
显著事件死亡          2
其他疾病/自然死亡     16
人口实际扣减          18
```

若重大事件产生 30 名显著死亡，而当天基线计划为 18 人：

```text
全因死亡             30
基线死亡             18
超额死亡             12
人口实际扣减          30
```

超额死亡不减少后续日期的基线死亡配额，也不由出生或迁移自动补偿。因此首周期的 1.9% 精确目标只适用于无超额死亡基线情景。

### 9.3 年龄—性别分配

- 显著死亡先按类别专属风险权重分配年龄和性别。
- 其余基线死亡按当日队列人口乘以单岁、分性别死亡率进行加权分配。
- 每次抽取后更新可用人口，任何队列不得变为负数。
- 若首选队列不足，按确定性的相邻风险队列顺序重新分配。
- 无法合法分配时抛出带状态路径的验证错误，不得静默丢失死亡。

死亡率基线必须同时校准到 6,600 人、男女寿命、婴儿死亡率和五岁以下死亡率。

## 10. 迁移模型

第一周期迁移配额：

| 类型 | 人数 | 年龄—性别特征 |
|------|-----:|---------------|
| 侨民永久回流 | 2,500 | 25–49 岁为主，包含配偶和儿童 |
| 外国移民流入 | 2,200 | 20–44 岁劳动人口为主，男性略多 |
| 马里文人永久移出 | 2,800 | 18–39 岁为主，包含学生、护士、教师和青年专业人员 |

迁入人口加入相应年龄、性别和生日月份桶；迁出人口从已有队列中扣除。任何迁出队列不得为负。

以下流动不改变常住人口：

- 圣诞和新年临时返乡。
- 旅游。
- 短期商务访问。
- 明确会在统计期内回国的季节劳工。

## 12. 内部状态

输出状态升级到 schema v3。`model_state.population` 至少保存：

- `baseline_version`
- `cohorts.male` 与 `cohorts.female`
- `birthday_buckets.male` 与 `birthday_buckets.female`
- 当前滚动周期的开始边界和结束日期
- 五类完整每日配额计划

状态验证必须保证：

```text
sum(cohorts) == sum(birthday_buckets) == state.population
state.demographics.population == state.population
```

## 13. 状态迁移

### 13.1 版本策略

- schema v1：无显式版本的旧状态。
- schema v2：现有天气、汇率、商品和 CPI 显式状态。
- schema v3：在 v2 基础上增加人口队列和公开人口对象。

迁移支持 v1 → v2 → v3 和 v2 → v3。schema v3 缺少人口状态时视为无效状态，不静默补齐。

### 13.2 v2 → v3

1. 读取当前顶层 `population` 和 `date`。
2. 加载固定版本的人口基线。
3. 若人口恰为 1,200,000 且日期为 2026-08-11，直接使用原始基线。
4. 其他人口使用基线比例缩放。
5. 使用最大余数法分配整数，使总和严格等于当前人口。
6. 同步缩放生日月份桶，并保持每个队列汇总一致。
7. 从当前状态日期的下一天开始建立新的 365 天周期。
8. 不伪造迁移前的出生、死亡或迁移历史。
9. 保留未知顶层字段和未知 `model_state` 字段。
10. 完整验证后才返回迁移状态；迁移本身不写文件。

## 14. 归档兼容

每日 JSON 归档自然包含 `demographics` 和完整 `model_state.population`。

第一版不为 SQLite `daily_summary` 增加人口列，也不新增人口统计表。为保持既有历史列含义：

- `daily_summary.deaths_total` 继续写入 `deaths_today.notable_total`。
- 各显著死亡分类列继续保持原含义。
- 全因死亡以 JSON 中的 `deaths_today.total` 和 `demographics.deaths_all_causes_today` 为权威。

后续统计局人口发布功能可以单独增加：

```text
population_annual_summary
population_age_groups
vital_events_monthly
```

这应作为独立设计和数据库迁移处理。

## 15. 错误处理与验证

模型必须拒绝：

- 男性或女性年龄组不是恰好 101 个。
- 生日月份桶形状不是 101 × 12。
- 人口出现负数、浮点人数或布尔值。
- 队列和生日桶不一致。
- 队列合计不等于顶层人口。
- 生育率、死亡率或迁移权重为负、非有限或无法归一化。
- 剩余配额小于零或大于周期总额。
- 当前日期与周期计划不一致。
- 显著死亡无法从任何合法队列扣除。
- 未知 `baseline_version`。
- JSON 中出现非有限数或不可序列化值。

错误信息必须带准确路径，例如：

```text
state.model_state.population.cohorts.male[34]
```

## 16. 测试策略

### 16.1 基线和单元测试

- 202 个单岁队列总和严格为 1,200,000。
- 生日月份桶逐队列汇总与单岁队列一致。
- 中位年龄约为 27.8 岁。
- 男女人口和 0–14、15–64、65+ 汇总正确。
- 5 岁组之和与总人口一致。
- 生育率曲线产生 27,500 名首周期新生儿。
- 新生儿性别比符合校准目标。
- 死亡曲线命中 6,600 人及寿命、婴儿和五岁以下死亡指标容差。
- 三类迁移年龄—性别结构符合设定。
- 显著死亡包含在全因死亡中，不重复扣减。
- 极端显著死亡形成超额死亡。
- 任何队列均不为负。
- 月度年龄推进不改变总人口，且同一人口不会在一个月推进多岁。

- 五类配额计划各自精确命中周期配额。

### 16.3 第一周期校准门

无超额死亡基线情景运行 365 天：

```text
出生             = 27,500
基线全因死亡     = 6,600
侨民回流         = 2,500
外国移民         = 2,200
永久移出         = 2,800
人口净增         = 22,800
期末人口         = 1,222,800
```

同时验证：

- 每日流量存在波动，不是固定常数。
- 年龄推进人数守恒。
- 所有状态可用严格 JSON 编码。
- 365 天运行时间适合纳入全量测试。
- 加入明确超额死亡后，期末人口只按超额死亡数低于基线目标。

### 16.4 长期测试

- 运行至 2035 年，TFR 接近 2.0。
- 侨民回流从疫情后高位逐步减弱。
- 中位年龄和老年人口占比呈合理演化。
- 长期增长率向约 0.8% 靠近，但使用合理区间，不要求精确等于 0.8%。
- 长期运行无负人口、非有限数、状态失配或不可接受的状态膨胀。

## 17. 第一版非目标

- 四岛、城市或行政区人口队列。
- 城乡内部迁移。
- 族裔和宗教人口队列。
- 公民与外国居民的存量拆分。
- 教育、职业、收入和劳动状态。
- 婚姻、家庭和住户结构。
- 残障人口。
- 疾病死因明细模型。
- 实时边境入出境数据。
- 个人级 Agent 模拟。
- 人口网站和可视化页面。

第一版只建立全国单岁年龄—性别人口地基和唯一、正确的人口会计。后续模型通过明确接口扩展，不把所有社会维度塞入人口核心模块。

## 18. 验收标准

实现完成必须同时满足：

1. 基线数据可追溯且所有校准断言通过。
2. schema v1/v2 可无损迁移到 v3。
3. 第一周期无冲击情景精确得到 1,222,800 人。
4. 所有死亡只扣除一次，超额死亡不会被其他配额抵消。
5. 顶层、公开对象、单岁队列和生日月份桶在每个 Tick 后一致。
6. 确定性、序列化恢复和随机流隔离测试通过。
7. 原有 P0 模型输出不因人口模型加入而漂移。
8. 全量测试、严格 JSON、编译和 365 天 dry-run 全部通过。
9. dry-run 与测试不改写运行状态、历史归档或 SQLite 数据库。
10. README 和模型目录准确标记人口增长与年龄结构模型的实现范围。

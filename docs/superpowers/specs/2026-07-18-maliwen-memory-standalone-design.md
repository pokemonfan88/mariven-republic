# 马里文国家记忆中心原生单文件版设计

## 目标

把现有 `maliwen-memory` React 网站重写为一个完全自包含的原生 HTML 文件，在不依赖 React、Vite、Tailwind、CDN 或网络连接的情况下，尽量一比一保留当前视觉、内容和交互。同时重做遇难者名录，使姓名符合马里文多族群世界观，避免现有随机拼接产生的生硬姓名。

最终交付文件为 `maliwen-memory/maliwen-memory-standalone.html`。现有 React 项目保持不变，作为视觉与内容参照。

## 交付边界

- 一个 HTML 文件，包含页面结构、样式、脚本和全部照片。
- 可通过 `file://` 双击离线打开，也可由普通静态服务器托管。
- 保留现有全部章节、文字、照片、档案说明、统计数字和页脚。
- 保留滚动进度、渐入效果、数字递增、名录搜索和岛屿筛选。
- 不新增后台、数据库、内容管理系统、用户提交或在线预约功能。
- 不修改现有 React 版本，也不删除其依赖或构建产物。

## 页面架构

页面继续采用当前长篇纪念叙事顺序：

1. 首屏：158,000 个名字
2. 独立之光
3. 裂痕
4. 第一次内战
5. 深渊：兄弟之战
6. 名录墙
7. 证词库
8. 灰烬上的房子
9. 宪法序言
10. 今日纪念与页脚

HTML 使用语义化的 `header`、`main`、`section`、`figure`、`blockquote`、`form` 和 `footer`。CSS 与 JavaScript 写在文档内部；照片转换为 `data:` URL，确保文件移动后仍能显示。

## 视觉与响应式

- 保留深蓝黑背景、暖金色强调、新闻衬线标题和档案照片的低饱和处理。
- 桌面端保持宽阔的长篇叙事节奏；移动端改为单列，缩小横向留白并保证筛选按钮可换行。
- 不加载在线字体，使用系统中文字体、Georgia 和 Times New Roman 回退组合。
- 所有可操作元素提供清晰的键盘焦点样式。
- 在 `prefers-reduced-motion: reduce` 下停用位移动画和数字跳动，内容直接显示。

## 原生交互

- 顶部进度条监听页面滚动并反映阅读进度。
- `IntersectionObserver` 控制章节渐入；不支持该 API 时直接显示内容。
- 数字统计进入视口后使用 `requestAnimationFrame` 递增；减少动态效果时直接显示最终值。
- 名录搜索同时匹配中文姓名、拉丁字母原名和岛屿。
- 岛屿按钮提供“全部、卡托拉、马卡迪岛、蒂莫岛、佩拉岛、鲁瓦岛”筛选。
- 搜索与筛选共同作用，结果数实时更新；无匹配项时显示明确提示。
- JavaScript 初始化失败时，核心叙事内容仍然可阅读。

## 遇难者名录重做

名录固定为 168 条策划记录，不再在浏览器中随机组合姓名。族群构成为：

| 族群 | 数量 | 占比 |
| --- | ---: | ---: |
| 原住民马里文人 | 134 | 79.8% |
| 印度裔马里文人 | 20 | 11.9% |
| 欧洲裔马里文人 | 6 | 3.6% |
| 华裔马里文人 | 4 | 2.4% |
| 其他太平洋族群 | 4 | 2.4% |

每条记录包含固定的中文姓名、原文拼写、岛屿、年龄、死亡日期和身份。页面显示格式为“中文姓名（原文拼写）”。这些字段直接写入页面数据，不随刷新变化。

### 原住民姓名规则

原住民姓名采用统一的马里文语风格，使用开放音节和南太平洋语感，但建立独立于现实国家的组合体系。常用音节包括 `lei`、`mere`、`semi`、`josefa`、`tala`、`vaka`、`rau`、`navi`、`tui`、`mako`、`loto`、`sere`。姓与名必须来自同一语言体系，例如：

- 莱萨·纳武阿（Leisa Navua）
- 约瑟法·塔乌马科（Josefa Taumako）
- 米蕾卡·瓦卡鲁阿（Mereika Vakarua）
- 塞米西·拉武托（Semisi Ravuto）

同一家族姓氏可以在同一岛屿少量重复，以体现家族和村落关系，但完整姓名不得重复。避免把中文、印度、欧洲或太平洋姓名部件跨文化随机拼接。

### 其他族群姓名规则

- 印度裔姓名采用完整且自然的南亚侨民命名方式，中文译名与拉丁拼写一一对应。
- 欧洲裔姓名采用英语姓名顺序，不混入原住民姓氏，除非人物被明确设计为跨族群家庭成员；本版不引入这类额外背景。
- 华裔姓名使用正常中文姓与名，并附汉语拼音。
- 其他太平洋族群采用一致的萨摩亚、汤加或邻近岛国风格姓名，不与原住民马里文姓名池交叉拼接。

年龄、日期、岛屿和身份分布保持多样，但不暗示某一族群天然属于某一交战方。平民应在各族群中占明显多数，符合网站现有叙事。

## 架空档案说明

页面在名录区和页脚以克制但可见的方式注明：马里文共和国及本页人物档案属于架空历史世界观，姓名和记录为虚构创作，不对应现实遇难者。说明不削弱纪念馆视觉，但必须避免读者把记录误认为现实史料。

## 验证标准

- 文件夹中只需复制该 HTML 文件即可完整浏览，无缺失图片或外部网络请求。
- 浏览器控制台没有错误。
- 页面包含全部 10 个叙事部分、10 张照片和 168 条固定名录。
- 168 个完整姓名唯一，族群数量与设计表完全一致。
- 每个姓名都有中文与原文拼写，刷新页面后内容和顺序不变。
- 中文、拉丁字母、岛屿搜索均返回正确结果；搜索与岛屿筛选可组合使用。
- 桌面和手机宽度下没有横向溢出，正文和按钮可读可操作。
- 关闭网络后通过本地文件打开，视觉与交互仍然正常。
- 开启“减少动态效果”后，所有内容立即可见且功能不受影响。

## 实施顺序

先把现有 React 页面内容映射为语义化 HTML，再建立原生 CSS 设计系统；随后策划并校验 168 条固定名录，实现原生交互，最后内嵌照片并进行桌面、移动端与离线验证。任何内容或视觉差异都以现有网站为基准逐项核对。

# 2026 年完整季度 GDP 国民核算模型设计

**日期：** 2026-07-18  
**状态：** 已批准，进入实现  
**范围：** 2025 年年度基准、2026 年季度生产法与支出法核算、每日潜在活动、季度结算、发布时滞、修订版本、主 Tick 集成和 schema v4 状态迁移

## 1. 目标

把当前固定的 `economy.gdp_growth_pct` 升级为可跨日持久化、可复现、可解释的季度国民核算模型。模型以生产法为权威总量，以支出法作独立核对，并为人均 GDP、财政、债务、贸易、就业、犯罪和公共卫生等后续模型提供统一经济底座。

模型必须同时满足以下要求：

- 2025 年名义 GDP 权威基准为 163.5 亿 MVL。
- 2025 年实际 GDP 增长率为 3.1%。
- 2026 年中央情景使用 2.4% 实际增长和 2.2% GDP 平减指数增长。
- 内部每天累计潜在经济活动，但公开 GDP 只按季度并在季末后 90 天发布。
- 生产法行业增加值、支出法恒等式和公开总量必须严格闭合。
- 真实外部数据只提供 2026 宏观路径和核算结构先验，不直接复制斐济经济规模。
- 新增 GDP 模型不得改变天气、汇率、商品、CPI、人口或既有事件的历史随机序列。

## 2. 已确认的核算口径

### 2.1 年度锚点

| 指标 | 2025 基准 | 2026 中央情景 |
|---|---:|---:|
| 名义 GDP | 16,350,000,000 MVL | 17,110,732,800 MVL |
| 实际 GDP 增长 | 3.1% | 2.4% |
| GDP 平减指数增长 | 项目基准 | 2.2% |
| 人口 | 1,200,000 | 使用完整人口模型的实际状态 |

2026 名义目标按以下公式形成：

```text
16,350,000,000 × 1.024 × 1.022 = 17,110,732,800 MVL
```

2026 年 2.4% 的增长路径来自 IMF 2026 年斐济 Article IV，只作为相近南太平洋岛国的宏观先验。马里文的规模、产业权重和冲击响应仍由本项目设定。

### 2.2 旅游口径修正

现有资料把游客总消费、旅游企业营业收入和旅游直接增加值混为同一指标。模型统一为：

- 110 万到访人次是包含邮轮访客的边境流量指标。
- 8.3 天和每日 680 MVL 用于估算游客境内总消费，年度约 62.084 亿 MVL。
- 旅游直接增加值固定为 16.35 亿 MVL，即 2025 GDP 的 10%。
- 游客消费中的进口食品、燃油、航空投入和其他中间消耗不得重复计入 GDP。

游客总消费是旅游和服务出口驱动量；旅游直接增加值才进入生产法 GDP。

### 2.3 生产法行业

生产法使用 13 个行业和一个产品税净额项目：

1. 农林渔业
2. 采矿采石
3. 制造业
4. 电力与供水
5. 建筑业
6. 批发零售
7. 住宿餐饮与旅游
8. 交通仓储
9. 信息通信
10. 金融保险
11. 房地产与商业服务
12. 公共管理、教育与卫生
13. 其他服务
14. 产品税净额

行业权重严格聚合为现有世界观四大类：第一产业及采矿 14.2%、工业 17.8%、服务业 57.4%、产品税净额 10.6%。所有行业值都以 MVL 保存，公开输出同时给出占比和增长贡献。

### 2.4 支出法

支出法使用以下恒等式：

```text
GDP = 居民消费 + 政府消费 + 固定资本形成
    + 存货变化 + 出口 - 进口 + 统计误差
```

2025 支出结构采用斐济统计局 2024 完整支出法的消费、资本形成和净出口比例作为先验，再按马里文既有贸易与旅游设定校准。出口和进口必须包含货物与服务；现有 14.8 亿 MVL 货物出口和 28.1 亿 MVL 货物进口只是其中的货物子集。

统计误差是显式项目，负责吸收两种核算方法在动态驱动后的微小差异。生产法是权威总量，统计误差调整后支出法必须精确等于生产法。

## 4. 运行架构

新增：

```text
sim-engine/engine/gdp_model.py
sim-engine/tests/test_gdp_model.py
sim-engine/scripts/build_gdp_baseline.py
```

### 4.1 模型接口

```python
gdp_step(
    current_date,
    previous_gdp_state,
    weather,
    cpi,
    commodities,
    exchange,
    population,
    baseline,
) -> (public_gdp, next_gdp_state, events)
```

`gdp_model.py` 负责：

- 加载并校验 GDP 基线。
- 从旧状态日期初始化季度台账。
- 计算每日行业实际产出和名义产出。
- 聚合并关闭季度。
- 生成支出法核对和统计误差。
- 管理待发布季度、正式发布和修订版本。
- 生成当前季度 nowcast、年度 nowcast 和公开 GDP 快照。
- 计算人均 GDP，但不修改人口权威账本。

`engine.py` 只负责模型顺序和公开状态装配，不包含 GDP 数学规则。

### 4.2 每日数据流

```text
天气、汇率、商品、CPI
          ↓
GDP 每日潜在活动与季度累计
          ↓
事件模型读取当日完整经济状态
          ↓
人口模型结算当日人口
          ↓
按期末人口和当日汇率刷新人均 GDP
```

GDP 在事件模型之前运行，使后续事件和犯罪模型能读取当日经济状态。人口结算后，只刷新 GDP 公开对象中的人口与人均值，不改变 GDP 台账。

## 5. 日度行业驱动

每个行业先取得基准日度实际增加值，再乘以有界驱动因子。驱动响应全部版本化保存在基线中。

| 行业 | 主要驱动 |
|---|---|
| 农林渔业 | 降雨、极端天气、全球糖价 |
| 采矿采石 | 全球金价、天气中断 |
| 制造业 | 全球糖价、油价、汇率、燃油成本 |
| 电力与供水 | 降雨、燃油成本 |
| 建筑业 | 降雨和气旋中断 |
| 批发零售 | CPI 实际购买力、人口 |
| 住宿餐饮与旅游 | 季节、天气、汇率、气旋风险 |
| 交通仓储 | 旅游活动、燃油成本、天气 |
| 信息通信 | 潜在趋势、人口 |
| 金融保险 | 潜在趋势、汇率波动 |
| 房地产与商业服务 | 潜在趋势、人口 |
| 公共服务 | 人口和既定政府路径 |
| 其他服务 | 居民需求和人口 |
| 产品税净额 | 名义消费与进口活动 |

驱动因子设置上下界，单日普通天气不会造成不合理的大幅 GDP 波动。气旋等重大灾害可以通过显式事件冲击接口在后续模型中增加额外损失；第一版不自行虚构灾害损失。

## 6. 季度、发布与修订

### 6.1 季度结算

- 每日实际和名义增加值累计到当前季度。
- 季末最后一个 Tick 关闭季度并写入 `quarterly_history`。
- 季度关闭后数值不可被后续普通 Tick 改写。
- 季节权重按行业分别保存，所有季度权重之和严格为 1。

### 6.2 官方发布

- 初值：季末后 90 天。
- 修订值：季末后 180 天。
- 最终值：季末后 365 天。
- 同一日期重复执行必须幂等，不生成重复版本或重复事件。
- 发布日之间，公开的 `latest_release` 保持最近一次已发布版本。

如果模型从 2026-08-11 的旧状态迁移：

- 2026-Q1 已关闭并已有初值。
- 2026-Q2 已关闭但尚未达到 90 天发布日。
- 2026-Q3 累计到迁移日期。

迁移不会为迁移日前的历史发布补发当日事件。

## 7. 状态与公开输出

### 7.1 schema v4

状态升级到 schema v4，并新增必需的 `model_state.gdp`。迁移链支持：

```text
v1 → v2 → v3 → v4
v2 → v3 → v4
v3 → v4
```

schema v4 缺少 GDP 状态时视为无效状态，不静默补齐。迁移函数可从当前日期和基线确定性初始化。

原有模型继续使用其原随机 schema：天气、汇率、商品、CPI、事件保持 v2；人口保持 v3；GDP 为确定性模型，不消耗其他随机流。

### 7.2 内部状态

`model_state.gdp` 至少保存：

- 模型版本和基线版本。
- 最后处理日期。
- 当前季度及其日度累计。
- 已关闭季度历史。
- 每个季度的初值、修订值和最终值发布版本。
- 最近公开版本。
- 当前年度累计和 nowcast 所需状态。

历史长度有明确上限；长期运行不允许状态无限膨胀。

### 7.3 公开对象

`economy.gdp` 至少包含：

- `latest_release`：参考季度、发布日期、版本、名义 GDP、实际同比、平减指数、行业贡献和支出法。
- `current_quarter_nowcast`：当前季度、截至日期、完成比例、实际与名义预测。
- `annual_nowcast`：年度名义 GDP、实际增长、平减指数和风险区间。
- `gdp_per_capita_mvl`、`gdp_per_capita_usd`、使用的人口和汇率。
- 数据是否正式、是否为模型估计、下一发布日期。

兼容字段 `economy.gdp_growth_pct` 更新为最近官方季度的实际同比；若尚无官方季度，则使用年度基准值。

## 8. 校验与错误处理

加载基线时拒绝：

- 非有限数、负 GDP 或无效比例。
- 行业权重不等于 100%。
- 季度季节权重不等于 1。
- 支出法不闭合。
- 2026 实际增长和平减指数与来源锚点不一致。
- 旅游直接增加值不等于 2025 GDP 的 10%。
- 外部来源缺少 URL、访问日期或提取值。

状态校验时拒绝：

- 当前季度与状态日期不匹配。
- 已关闭季度重复或顺序错误。
- 行业合计不等于生产法总量。
- 支出恒等式不闭合。
- 发布日在季度结束之前。
- 公开 GDP 与内部最近发布版本不一致。
- 非 JSON 值、NaN、无穷大或布尔值冒充数字。

所有错误都包含精确字段路径。

## 9. 测试与验收

### 9.1 基线测试

- 2025 名义 GDP 精确等于 16,350,000,000 MVL。
- 四大产业聚合精确等于 14.2%、17.8%、57.4% 和 10.6%。
- 2026 无冲击情景实际增长精确等于 2.4%，平减指数增长精确等于 2.2%。
- 生产法和支出法逐年、逐季度闭合。
- 旅游总消费与直接增加值口径分离。
- 已提交基线可由生成器逐字节复现。

### 9.2 模型测试

- 每日 Tick 不修改输入且结果可复现。
- 序列化后恢复运行与连续运行完全一致。
- 普通天气驱动有界，气旋风险降低受影响行业。
- 糖价、金价、油价和汇率只影响声明的行业。
- 季末只关闭一次季度。
- +90、+180、+365 天分别产生初值、修订值和最终值。
- 发布日之间官方值保持不变。
- 迁移到 2026-08-11 时，Q1 已发布、Q2 待发布、Q3 正在累计。
- 人均 GDP 始终使用当日人口和汇率。

### 9.3 集成回归

- schema v1、v2、v3 都能迁移到 v4。
- 主 Tick 公开 `economy.gdp` 并持久化 `model_state.gdp`。
- GDP 加入前后的天气、汇率、商品、CPI、人口和显著事件输出保持不变。
- 全量测试、365 天 dry-run、严格 JSON 序列化和 `compileall` 全部通过。

## 10. 第一版明确不包含

- 月度 GDP 官方发布。
- 省级、城市或岛屿 GDP。
- 企业级 agent-based 生产网络。
- 完整投入产出表和社会核算矩阵。
- 财政、债务、就业、旅游和贸易的独立子模型。
- 自动抓取尚未发布的 2025 斐济完整生产法数据。

第一版只建立全国生产法权威账本、支出法核对、季度发布制度和与现有 P0 模型的明确接口。后续模型通过版本化输入扩展，不把所有经济子系统塞入 GDP 核心。

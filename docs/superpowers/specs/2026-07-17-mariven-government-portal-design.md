# 马里文共和国政府门户设计

**日期：** 2026-07-17

**状态：** 已确认，待实施计划

**范围：** 响应式静态政府网站，共五个页面；英文与中文完整双语；固定日期国家信息快照；共享视觉、导航、搜索、筛选和无障碍规范

## 1. 目标

为马里文共和国建立一个可信、可用、具有本国身份的政府公共信息总入口。网站既要帮助公民找到服务和机构，也要承载政府新闻、公共公告、国家概况与日常公共状态。

设计必须满足：

- 第一眼像一个真实运转的南太平洋岛国政府门户，而不是旅游宣传页、智慧城市模板或 SaaS 落地页。
- 信息丰富但层级明确，不依赖“卡片套卡片”制造版面。
- 体现马里文的国徽、国旗色、格言、树皮布工艺、四岛社会与议会制共和国结构。
- 英文为默认完整界面，中文为完整对应界面；现阶段不制作不成熟的马里文语正文。
- 五个页面在桌面、平板和手机上均可独立使用。
- 不依赖服务器、前端框架或运行时网络资源。
- 页面数据明确为 2026-07-17 的静态国家信息快照，不冒充实时数据。
- JavaScript 关闭、图片丢失或筛选无结果时，核心信息仍可获得。

## 2. 非目标

首版不包含：

- 真实登录、身份认证、支付、表单提交或在线审批。
- 新闻详情页、服务办理详情页或单独的总统个人网站。
- 与模拟引擎、天气模型、CPI、汇率或人口模型的动态连接。
- 随机生成或随刷新变化的国家数据。
- 完整马里文语界面。
- 后台内容管理系统、构建服务器或部署平台绑定。
- 用旅游图库、海滩大图、Emoji 或其他国家徽章代替正式国家资产。

## 3. 现实政府网站研究结论

设计参考来自南太平洋政府官方站点，但不复制任何单一网站：

- 斐济政府门户以新闻、演讲、法律、预算、招标、职位和机构目录构成“政府新闻总台”。参考：<https://fiji.gov.fj/Home>
- 萨摩亚 One Government Portal 以服务和机构目录为主，驾驶执照、护照、民事登记、水电等按公民任务组织。参考：<https://mygov.gov.ws/>、<https://mygov.gov.ws/services/>
- 瓦努阿图总理府网站通过国徽、多语言、密集公共发布和部门入口形成强烈真实机构感，但信息层级较拥挤。参考：<https://pmo.gov.vu/en/>
- 库克群岛总理办公室突出本地语言政府名称、国徽、领导人纪实摄影以及气候和应急政策。参考：<https://www.pmoffice.gov.ck/>
- 斐济数字服务页面直接说明费用、办理时长、线上或线下渠道，是服务可信度的重要来源。参考：<https://bdm.digital.gov.fj/>

马里文采用“公民公报”路线：以政府公报和当代新闻编辑设计为主体，吸收数字门户的搜索效率与国家档案的庄重感。

## 4. 页面范围与导航

首版包含五个独立页面：

| 页面 | 建议文件 | 主要职责 |
|------|----------|----------|
| Home | `index.html` | 国家公共信息总入口 |
| Services | `services.html` | 政府服务搜索与分类目录 |
| Government | `government.html` | 国家领导人、宪政机关和机构名录 |
| News & Notices | `news.html` | 新闻、公告、声明、招标及紧急信息 |
| About Mariven | `about.html` | 国家概况、历史、文化、四岛和国家象征 |

所有页面共享：

- 公共状态栏。
- 正式国家页眉和五项主导航。
- 英中语言切换。
- 全站搜索入口或当前页面搜索。
- 国家页脚。
- 同一套色彩、排版、图标、焦点和交互规范。

主导航英文为 `Home`、`Services`、`Government`、`News & Notices`、`About Mariven`；中文对应为“首页”“政府服务”“政府机构”“新闻与公告”“关于马里文”。当前页面必须有清楚的非颜色单一提示。

## 5. 全站外壳

### 5.1 公共状态栏

状态栏位于最顶部，信息精简为：

- 国家信息快照日期：17 July 2026 / 2026年7月17日。
- 卡托拉市天气摘要。
- 国家紧急热线。
- 当日渡轮或灾害状态。
- `English / 中文` 语言选择。

状态栏不是实时行情跑马灯。紧急状态使用砖红色；普通状态使用海洋蓝或中性色。

### 5.2 正式页眉

页眉包含独立绘制的国徽、`Government of the Republic of Mariven`、`马里文共和国政府` 和格言 `Filemu, Saoloto, Tasi`。马里文语只用于已确认的国家名称、问候与格言等识别文字。

桌面导航保持横向、克制和正式；移动端使用明确的菜单按钮及纵向导航。页眉可在滚动后保持可达，但不得以大面积阴影或玻璃模糊形成互联网产品感。

### 5.3 国家页脚

页脚包含：

- 部委和机构目录入口。
- 政府办公时间、通信地址、总机和紧急联系。
- 无障碍、隐私、档案、信息公开、免责声明与网站反馈。
- 国家信息快照日期与页面版本。
- 国旗色细线和局部树皮布几何纹样。

## 6. 首页设计

首页从上到下按以下顺序组织。

### 6.1 主新闻滚动屏

首屏展示三条政府重点新闻，每条包含：

- 一张政府纪实主图。
- 新闻类别和发布日期。
- 清晰标题和一段简短摘要。
- 指向新闻列表或静态演示目标的明确链接。

滚动屏自动切换间隔约 7 秒，提供上一条、下一条、页码和暂停控制。鼠标悬停、键盘聚焦、用户手动操作或系统开启 `prefers-reduced-motion` 后停止自动播放。无 JavaScript 时显示第一条完整主新闻。

动画只允许淡入或短距离水平切换，不使用视差、放大镜头、光效或商业广告式过渡。移动端降低图片高度，并通过图片焦点设置保护关键人物和标识。

### 6.2 政务搜索与高频服务

滚动屏下方立即提供“你需要办理什么？”搜索和六项高频服务，避免首屏只有新闻。

高频服务建议包括：护照、出生证明、企业注册、驾驶执照、土地记录和医疗服务。每项使用统一线性 SVG 图标、短标题和真实办理状态，不使用 Emoji。

### 6.3 国家状态横带

显示人口、CPI、货币汇率以及天气或卫生指标。标题固定为 `National snapshot · 17 July 2026` / `国家信息快照 · 2026年7月17日`。

所有数值为编辑确认后的静态值；不使用计数动画，不写“实时”，也不在刷新页面时变化。

### 6.4 服务捷径

使用两列文字目录、章节编号、细分隔线和少量状态标签，而不是大面积圆角卡片。每条服务显示：

- 服务名称。
- 主管机构。
- `Online`、`Guide only`、`Office visit required` 或 `Unavailable` 等状态。
- 一个语义明确的动作链接。

### 6.5 新闻与公共公告

滚动屏之外的首页新闻区域采用一条主新闻、两条次新闻和一列紧凑公共公告。重要新闻可配图，普通公告仅用日期、编号、标题和来源机构。

该区域不得重复滚动屏的三条标题；应展示较低优先级但更实用的发布，例如燃油价格、学校安排、卫生提醒、选民登记或道路施工。

### 6.6 四岛公共信息

不使用地图、岛屿轮廓或四张旅游卡片。使用四条编辑式岛屿条目，桌面端两列、移动端纵向排列：

- Makadi Island / 马卡迪岛：910,000 人。
- Timo Island / 蒂莫岛：150,000 人。
- Pela Island / 佩拉岛：85,000 人。
- Ruwa Island / 鲁瓦岛：55,000 人。

每条包含行政中心、天气摘要、交通状态、地方政府入口和一项生活相关公共信息。仅一至两条可配小幅纪实照片，其他依靠排版和分隔线建立节奏。

四岛人口以 `sim-engine/data/nation_profile.json` 的全国一致口径为首版权威值。七省页面不显示存在口径冲突的省级人口数字，只列名称、行政中心和职能，避免首版内部矛盾。

### 6.7 政府与透明度入口

以正式链接组展示预算、法律、公报、议会记录、招标、职位、开放数据和信息公开。该区域可使用深蓝底色或暖灰底色形成页面收束，不使用八张同款卡片。

## 7. 政府服务页

页面顶部是大型服务搜索，而不是宣传横幅。搜索下方提供两种浏览方式：

- 按对象：个人与家庭、企业、访客与移民、非营利组织。
- 按人生事件：出生、教育、就业、创业、住房、交通、医疗、退休、身后事务。

服务条目以可扫描的列表或表格式行呈现，字段包括：

- 服务名称和一行说明。
- 主管机构。
- 预计办理时间。
- 费用或 `No fee`。
- 线上、仅指南、需到办公室或暂未开放的状态。
- 查看指南或前往机构的动作链接。

首版不制作虚假在线表单。演示状态必须使用诚实文案，如 `View application guide`、`Visit an office`、`Online applications not yet available`。

**总理托马斯·马卡里 / Prime Minister Thomas Makari**

### 8.2 宪政结构与名录

领导人之后依次展示总统办公室、总理与内阁、78 席一院制议会、司法机关和酋长大会的关系。

机构名录分为部委、独立宪政机构、国有公共事业和四岛地方政府。使用可搜索的正式名录行；每行包括机构名称、职责摘要、负责人、电话、办公地点和网站状态。部长头像只在确有必要的领导信息中使用。

## 9. 新闻与公告页

顶部保留一条编辑精选新闻，其余条目按发布日期倒序排列。分类为：

- Government News / 政府新闻。
- Public Notices / 公共公告。
- Cabinet Decisions / 内阁决定。
- Speeches & Statements / 演讲与声明。
- Tenders & Vacancies / 招标与职位。
- Emergency Information / 紧急信息。

提供类别和年份筛选。列表采用日期栏、标题、摘要和来源机构组成的报纸式结构；只有重点新闻配图。首版不制作独立新闻详情页，条目使用展开摘要或静态演示链接。

公共公告编号、发布日期和来源机构必须比装饰标签更醒目。紧急信息采用砖红色边线和明确图标，但不闪烁。

## 10. 关于马里文页

该页以正式国家概况为主，历史文化叙事穿插其中：

1. 国家简介与关键事实。
2. 四岛与七省。
3. 人民、语言和宗教。
4. 政体、宪法和国家机构。
5. 从公元前约 1500 年航海定居、英国保护领、1970 年独立、1987 至 1989 年内战与战后和解到当代的简明时间线。
6. 国旗、国徽、国歌、国花和国鸟。
7. 经济、气候与国际关系概览。
8. 宪法、国家统计、公报和开放数据等进一步阅读入口。

该页允许比其他页面更具叙事性，但保持政府出版物口吻。影像使用档案照片、仪式、聚落、公共建筑和日常生活，不使用“珊瑚天堂”“黄金之岛”等旅游广告文案。

关于页只解释总统与总理的宪政关系，不重复政府机构页中的完整领导人介绍。

## 11. 视觉系统

### 11.1 色彩

- 深海军蓝：页眉、主标题和权威信息。
- 暖象牙白：主要页面底色。
- 森林绿：国土、公共服务和普通正向状态。
- 暗金色：国徽线条、章节编号、重点分隔。
- 海洋蓝：天气、交通和一般公共状态。
- 砖红：仅用于紧急警报和严重中断。

色彩来源于国旗设定的深蓝 `#00205B`、绿 `#009A44` 和金 `#FFD100`，但网页使用经过对比度校正的较深文本版本。禁止大面积蓝紫渐变、霓虹光、玻璃拟态和发光边框。

### 11.2 字体

- 英文正式标题：本地打包或系统回退的公共机构气质衬线字体。
- 英文正文与导航：清晰的人文无衬线字体。
- 中文标题：宋体风格字体栈。
- 中文正文：黑体风格字体栈。
- 日期、公告编号和部分数字：等宽或窄体数字样式。

字体必须本地可用或使用稳健系统字体栈，不从在线字体服务加载。

### 11.3 版式

- 桌面端采用不对称十二栏网格。
- 区块依靠字号、留白、底色和细线分隔。
- 圆角仅用于搜索框、按钮和少量状态标签。
- 阴影极少使用，仅在必要的浮层或移动菜单中出现。
- 可点击文本通过下划线、颜色、箭头和焦点状态表达。
- 树皮布纹样只用于页眉细边、章节标记和页脚，不铺满背景。
- 图标使用同一套线性 SVG，不使用 Emoji。

## 12. 影像与国家资产

主新闻影像应呈现领导人、社区会议、码头、学校、诊所、农业、基础设施和灾害响应现场。照片保留自然光、环境背景和适度现场细节，不追求电影级光效或过度磨皮。

禁止：

- 将热带海滩图库作为首页主视觉。
- 同一图片重复用于多个栏目。
- 明显生成式人物、错误手部、虚构文字或不一致制服。
- 用 Emoji 或其他国家国徽代替马里文国家资产。

国徽、国旗和线性图标应作为本地 SVG 资产独立绘制。国徽遵循现有设定：塔普山、四星、甘蔗与椰叶、双体帆船、翠鸟、海豚和格言绶带。

## 13. 双语策略

- 英文是默认完整界面。
- 中文提供完整内容对应，不只翻译导航。
- 语言选择存入 `localStorage`，跨五页延续。
- 切换语言时同步更新文档 `lang`、页面标题、导航、表单提示、按钮和无障碍标签。
- 无 JavaScript 时英文内容正常显示。
- 马里文语只用于 `Republika Mariven`、`Filemu, Saoloto, Tasi`、`Bula` 等已确认识别文字，不提供第三个切换项。

实现时双语正文必须在 HTML 中可读取，不能由 JavaScript 远程获取。隐藏语言不应进入键盘焦点顺序或被屏幕阅读器重复朗读。

## 14. 静态技术架构

建议目录：

```text
gov.mv/
├── index.html
├── services.html
├── government.html
├── news.html
├── about.html
└── assets/
    ├── css/
    │   └── site.css
    ├── js/
    │   └── site.js
    ├── icons/
    └── images/
```

技术约束：

- 使用语义化 HTML、原生 CSS 和少量原生 JavaScript。
- 不使用 Tailwind、Bootstrap、Font Awesome、在线字体 CDN 或前端框架。
- 核心内容直接存在于各 HTML 文件；JavaScript 只增强轮播、语言、搜索、筛选和移动导航。
- 五页可通过本地静态服务器和任意静态托管访问。
- 不要求构建步骤；如后续为减少共享页眉重复引入构建工具，必须保持输出为完整静态 HTML，且不属于首版范围。

## 15. 交互与状态

首版交互包括：

- 三屏主新闻轮播及暂停、前后和页码控制。
- 移动导航。
- 英中语言切换。
- 服务、机构和新闻的浏览器内搜索与筛选。
- 新闻摘要展开。
- 清楚的当前导航和焦点状态。

搜索采用当前页面已有内容匹配，不跨站抓取。筛选无结果时显示结果数、清除条件按钮和相关目录入口。

所有演示链接按动作分类：

- `View guide / 查看指南`。
- `Contact agency / 联系机构`。
- `Visit office / 前往办公室`。
- `Online service unavailable / 在线服务暂未开放`。

不得让 `#` 链接看起来像可完成的真实办理流程。

## 16. 降级与错误处理

- JavaScript 关闭：英文正文、五页导航和第一条主新闻仍可阅读；搜索和筛选退化为完整列表。
- 图片加载失败：显示固定比例中性背景、资产说明和可读替代文字，不显示破图图标。
- 搜索无结果：提供清除筛选和目录导航。
- 本地存储不可用：语言切换在当前页面仍工作，跨页恢复退化为英文默认。
- 轮播脚本异常：第一条新闻保持静态显示，控制按钮不遮挡正文。
- 长标题：允许自然换行，不使用省略号隐藏关键信息。

## 17. 响应式与无障碍

重点验收宽度为 1440、1024、768 和 390 像素。

- 桌面：不对称十二栏编辑网格。
- 平板：新闻、服务和状态区域重组为两列或主次结构。
- 手机：滚动屏降低高度；新闻、服务、四岛条目和名录全部纵向排列；公共状态栏允许换行或折叠次要信息。
- 不允许横向页面滚动、导航遮挡正文或中文标题单字孤行。

无障碍要求：

- 跳至正文链接和语义化地标。
- 正确的标题层级。
- 所有图片的有意义替代文字。
- 可见键盘焦点和完整键盘操作。
- 触控目标最小约 44 × 44 像素。
- 文本与背景对比度达到 WCAG AA。
- 轮播有可读名称、当前项状态和暂停机制。
- `prefers-reduced-motion` 下禁用自动轮播和非必要过渡。
- 不能仅依靠颜色传达警报、当前页面或办理状态。

## 18. 内容语气

英文和中文都使用清楚、克制、可核验的政府出版物语气：

- 先说明事实、日期、主管机构和公众需要做什么。
- 避免“世界级”“天堂”“前所未有”等宣传词。
- 保留真实政府信息的具体性：费用、办理时间、办公地点、服务限制、资料日期。
- 历史页面承认殖民、内战与和解，不用空泛英雄叙事覆盖复杂历史。
- 新闻标题使用句式大小写或正常中文标题，不全部大写。

## 19. 验收标准

实施完成后必须验证：

1. 五个页面均可访问，主导航与页脚一致。
2. 英文默认、中文完整，跨页语言偏好有效。
3. 轮播可自动播放、暂停、前后切换、页码切换和键盘操作。
4. 减少动态效果时轮播不自动播放。
5. 服务、机构和新闻搜索及筛选正常，空结果状态可恢复。
6. JavaScript 关闭后核心内容和导航仍可使用。
7. 图片缺失时出现可读占位和替代文字。
8. 1440、1024、768、390 像素下无横向溢出和内容遮挡。
9. 所有交互可用键盘完成，焦点清楚，色彩对比合格。
10. 无明显控制台错误、断链、错误语言混排或重复朗读。
11. 页面不出现卡片套卡片、Emoji 图标、旅游宣传式首屏、蓝紫渐变、玻璃拟态或过度动画。
12. 总统和总理职责表述符合议会制共和国设定，不形成个人崇拜式呈现。
13. 所有国家状态数据标注 2026-07-17 快照日期，不声称实时。

## 20. 后续扩展边界

待世界观与模拟数据成熟后，可以在不改变页面结构的前提下：

- 将国家状态横带和四岛状态改为读取本地 JSON。
- 接入天气、CPI、汇率、人口和公共事件快照。
- 增加马里文语完整界面。
- 增加服务详情、新闻详情和真实在线办理系统。
- 将公共状态栏升级为经审核的紧急信息源。

首版必须先建立可信的静态信息架构和视觉规范，动态化不得提前引入不完整或随机数据。

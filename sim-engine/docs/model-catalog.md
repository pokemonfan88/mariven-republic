## P0 运行语义

当前 P0 已实现范围为天气、汇率、商品价格、CPI、人口和 GDP，六者均接入 `engine/engine.py` 的每日主 Tick。CPI 在每月 15 日发布并在发布日之间保持最近的官方值；商品价格保留实际来源月份，并以 `source_month`、`staleness_days` 和 `is_stale` 暴露数据陈旧度。人口模型使用 schema v3 的男性/女性 0–100+ 单岁队列、生日月份桶和独立确定性随机流，显著死亡由事件模型分类、全因死亡只在人口账本扣除一次。GDP 使用 schema v4 的日度行业累计、季度生产法权威账本、支出法核对以及+90/+180/+365天发布版本。

```bash
cd sim-engine
python engine/engine.py --days 1
python engine/engine.py --days 30
python engine/engine.py --days 365 --dry-run
python -m unittest discover -s tests -p "test*.py" -v
```

`--dry-run` 只执行模拟并输出结果，绝不写入状态文件、JSON 历史归档或 SQLite 事件库。

## 可行性总结

| 难度 | 数量 | 说明 |
|------|------|------|
| ⭐ 极简单 | 9 | 除法、会计、线性趋势。周末写完。 |
| ⭐⭐ 适中 | 25 | Markov/Gamma/正弦/泊松。纯Python，不依赖外部数据。每一类约半天。 |
| ⭐⭐⭐ 需要外部数据 | 20 | IBTrACS/USGS/ICE糖期货/布伦特原油/世界银行。需下载+清洗+打包JSON。每个约1天。 |
| ⭐⭐⭐⭐ 复杂 | 20 | 多模型联动/GDP结构/民调卡尔曼/联盟博弈。需要设计+测试。每个约2-3天。 |
| ⭐⭐⭐⭐⭐ 极难 | 10 | Agent-based仿真/社交媒体/收入分配/复杂网络。需要专项研究与大量调试。放在最后。 |

**总评估**：
- **75% 的模型（56个）** 是 ⭐⭐ 或以下——纯 Python + 基础数学，一个人几周能做完
- **最难的部分**（政治民调/基尼系数/社交媒体）可以先用简化版替代，后期迭代
- 外部数据源全部公开免费——IBTrACS、World Bank Pink Sheet、USGS、ICE——不存在数据获取障碍
